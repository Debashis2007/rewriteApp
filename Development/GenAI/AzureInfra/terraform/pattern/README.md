# Terraform Patterns

This directory contains reusable patterns and best-practice templates for common Azure infrastructure scenarios. Each pattern demonstrates how to combine modules and resources to solve real-world problems or implement architectural best practices.

## Example Patterns

- **hub-spoke-networking.tf**: Pattern for secure, scalable networking using a hub-and-spoke topology.
- **secure-workload.tf**: Pattern for deploying a secure workload with Key Vault, Storage, and RBAC.
- **compliance-landing-zone.tf**: Pattern for a compliant landing zone with policy, audit, and evidence storage.

## How to Use

1. Copy or reference these pattern files in your own Terraform projects.
2. Customize variables and resource names as needed.
3. Use these as starting points for your own infrastructure patterns.

---

Add new patterns as your architecture evolves or as you identify reusable solutions!
